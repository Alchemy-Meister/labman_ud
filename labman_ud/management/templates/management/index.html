Management site


This site is intended to perform management tasks avoiding the command line. The different actions performed in this section are only viewable by registered users, and its actions modify directly the data stored. Please be cautious before taking any action that could lead to information loss.


Similar person names
 This task will check all the person names in the system, trying to disambiguate those names with a relevant similarity ratio. The following buttons will filter the similarities with a ratio above the indicated threshold. For each similarity tuple, clicking on the button before each name will make that the official name for that person, making the other one an alias. If the user prefers to ignore this similarity (they address clearly two different persons), it will not be shown in later iterations. Clicking on the reset button will bring back all the ignored similarities.


Special tags list
When adding tags to publications on Zotero, there are some special tags available which will lead to further enhance the data about it. These tags will automatically be processed by labman_ud when the zotero extractor is launched, and will not be listed amont the related tags of publications. The tags which can be used are those in bold font, at the left side of both lists.

Tags management
 Tags address those topics projects, publications and news cover. At first, the whole list of tags is alphabetically displayed. Clicking on the edit button before each tag, takes the user to a page where the tag text and its parent tag (for topic hierarchy purposes) can be edited. This page also displays information about the entities within labman_ud that make use of that tag. Setting an already existing tag name will update all the entities to use the same tag, which is usefull to avoid topic redundancy.


Parse publications
 To ease the process of extracting information about publications, labman_ud relies on zotero, a tool to collect, organize, cite and share research sources among peers. To use the zotero extractor, please register a new application in order to get the OAuth credentials. Once done, fill the ZoteroConfiguration model in the admin site to have everything ready for the parser.
